> Статус: **experimental, вне semver-гарантии 1.0** (см. B3 в [ROADMAP-INTERNAL](./ROADMAP-INTERNAL.md)). Релиз — 1.x.

## Задача

Универсальная визуализация сетей связей: entity-графы (пример: связи Питера Тиля — личные, рабочие, инвестиции), коммерческие связи агентов, блокчейн-адреса. Текущий `Graph.js` — чистый d3-force, на реальных данных даёт нечитаемый клубок; это фундаментальное свойство force-directed раскладки, а не баг настройки сил.

## Ключевые принципы

1. **Никогда не рисуем весь граф — рисуем результат запроса.** Раскладка не «открывает» структуру силовой симуляцией, а кодирует известную семантику (степень удалённости, тип связи, найденный путь).
2. **«Сила связей» не задаётся — она раскладывается на две вычислимые вещи:**
   - *вес ребра* — наблюдаемые данные домена (число совместных инвестиций, объём транзакций, контракты) — приходит из источника данных;
   - *важность узла* — вычисляется метриками центральности: degree/eigenvector («у кого больше сильных связей» → Трамп), betweenness («через кого проходят пути» → незаметный ассистент-брокер), closeness. Брокеров глазом на картинке не видно — их находит только расчёт; поэтому аналитический слой первичен, картинка производна.
3. **Зум — семантический, не геометрический.** Отдаление не рисует десятый круг поузлово (это миллионы узлов и возврат хаоса), а переключает вид ego → cluster: дальние уровни агрегируются («PayPal-мафия, 34 узла», «+142 связи»), раскрываются при приближении/клике.

## Три слоя

| Слой | Что делает | Чем реализуется |
|------|-----------|-----------------|
| **Модель + алгоритмы** (headless) | Хранит накопленный граф, считает BFS, пути, центральности, сообщества | **graphology** (+ модули: shortest-path, metrics/centrality, communities-louvain). Чистые функции, tree-shakeable, MIT, без своего рендеринга — не конфликтует с темизацией Rare |
| **Раскладки** | Чистые функции «подграф → координаты», детерминированные | Свои, небольшие: сетка с категориями по углам (ego), слоистая (path), агрегатная (cluster). От полярной ego-раскладки отказались по итогам экспериментов — дуги дают кашу на плотном первом круге |
| **Рендеринг** | SVG, тема, тултипы, легенда | Существующий пайплайн RareCharts (`core/Chart.js` и далее), без изменения подхода |

Клиентская модель — не техническая деталь, а часть UX: она **накапливает** всё, что юзер «нагулял» (Тиль → Цукерберг → ...), повторные заходы не дёргают базу, локальные расчёты (центральности, пути внутри загруженного) работают по накопленному.

## Три вида (все — проекции одного графа)

- **Ego** — окрестность узла на глубину k (BFS, k — параметр, не константа). Эго закреплён в центральной клетке регулярной сетки; категории якорятся по углам/краям и заполняют клетки от дальнего угла к центру, так что удалённость от центра ≈ степень рукопожатий. Рёбра — с лёгким равномерным изгибом (прямые на сетке проходят сквозь коллинеарные узлы).
- **Path** — «как связаны A и B» (Тиль ↔ индонезийский фонд): k кратчайших путей, слоистая раскладка слева направо, колонки = хопы, как схема маршрутов. Хаос невозможен по построению.
- **Cluster** — обзорная карта: сообщества (Louvain) схлопнуты в мета-узлы. Единственный вид, где допустима force-подобная раскладка.

Переход ego ↔ cluster = семантический зум (см. принцип 3).

## Интерактивность (закладывается сразу, не «потом»)

- **Клик по узлу = перецентровка**: чарт запрашивает окрестность нового эго и перерисовывает тот же вид — бесконечная «прогулка» по графу.
- **Сохранение идентичности узлов** при перецентровке: общие узлы (старый эго остаётся в окружении нового) анимированно переезжают, а не перерисовываются с нуля — иначе юзер теряет ориентацию на каждом шаге.
- **Focus + context**: по умолчанию видны только связи с эго; рёбра второго круга появляются при hover/клике, остальное затухает. Даёт не меньше половины читаемости — перегрузка рёбрами разрушает любую раскладку.

## Требования к источнику данных

База — вне зоны ответственности библиотеки; библиотека принимает подграфы. Но контракт запросов фиксируем — минимум три:

| Запрос | Для какого вида | Примечание |
|--------|-----------------|------------|
| `neighbors(node, depth, filters)` | ego | фильтры по типу связи |
| `paths(a, b, k)` | path | **обязательно серверный**: путь между произвольными узлами не посчитать на клиенте — у клиента загружены только нагулянные окрестности |
| `aggregates(node, depth)` | cluster / семантический зум | количества и кластеры без деталей, лёгкий запрос |

## Набросок API

```js
const graph = new RareCharts.Graph('#el', {
  view: 'ego',                    // 'ego' | 'path' | 'cluster'
  dataSource: {                   // адаптер к бэкенду («source» занят подписью-атрибуцией в футере)
    neighbors: (node, opts) => Promise,
    paths: (a, b, opts) => Promise,
    aggregates: (node, opts) => Promise,
  },
  depth: 2,                       // ego: глубина BFS
  groupBy: 'relation',            // ego: секторы по типу связи
  onNodeClick: 'recenter',        // или callback
});
graph.focus('peter-thiel');       // ego view
graph.connect('peter-thiel', 'swiss-startup'); // path view
graph.overview();                 // cluster view
```

Форма данных — узлы/рёбра с типами: `{ nodes: [{ id, label, group, ... }], links: [{ source, target, type, weight }] }` (совместимо с текущим Graph.js, добавляются `type`/`weight`).

## Этапы (каждый самоценен)

1. **Модель + ego view** — graphology-прослойка, полярная раскладка, focus+context, перецентровка кликом. Уже пригодно для entity-кейсов.
2. **Path view** — серверный `paths()`, слоистая раскладка.
3. **Cluster view + семантический зум** — Louvain, агрегаты, переход ego ↔ cluster.
4. (Опционально, позже) адаптер сторонних раскладок (graphology-layout-forceatlas2 / elkjs) для больших произвольных графов — рендеринг не меняется.

## Что осознанно НЕ делаем

- **Не берём графовый фреймворк** (Cytoscape.js, Sigma.js, vis-network): они забирают рендеринг и стилизацию — конфликт с темой Rare Styles, лишний вес для CDN-библиотеки; их сила (большие произвольные графы) — не наш кейс. Заимствуем только headless-алгоритмы (graphology) и, при нужде, layout-функции.
- **Не рисуем полный граф** и не пытаемся починить текущий force подбором параметров сил.
- **Не тащим в 1.0** — класс experimental, живёт вне заморозки API (ROADMAP-INTERNAL, B3).
