## 1.0 = заморозка API, а не «больше фич»

Главный смысл мажорной версии по semver: публичная поверхность (имена опций, форма данных, имена классов) замораживается, и внешние проекты могут на неё полагаться. Библиотека уже грузится по CDN на боевых проектах — фактически контракт уже действует, но формально он ещё Beta. Всё ниже — про то, чтобы к 1.0 этот контракт был надёжным и проверяемым.

Новые типы графиков на 1.0 **не требуются** — они аддитивны и спокойно выходят в 1.x.

## Блокеры 1.0

### B1. Нет тестов
**Где:** весь `assets/charts/src/` (~5 400 строк), 0 тестов.
**Проблема:** шкалы, форматирование, клиппинг осей, кросс­хейр, адаптеры данных — без единого теста. После заморозки любой рефактор ломает потребителей вслепую; semver-обещание нечем подкрепить.
**Решение (зафиксировано):** один раннер — **Vitest + jsdom**. Одна девзависимость, один запуск (`npm test`). Несколько разных инструментов не нужны.
**Минимум к 1.0:**
- юнит-тесты на чистые функции: `core/utils.js`, `core/seriesPath.js`, адаптеры `adapters/index.js` (`fromJson`/`fromCsv`/`fromApi`/`fromArray`);
- smoke-тесты рендера в jsdom (каждый класс инстанцируется + `.setData()` без исключений).
**Релиз:** 0.9.9.

### B2. Противоречие в лицензии  **[parallel]**
**Где:** `assets/charts/LICENSE` и `assets/charts/README.md` → **MIT**; `package.json` → `"license": "ISC"`.
**Проблема:** библиотеку встраивают в коммерческие продукты; юридическая неоднозначность в момент 1.0 — риск доверия.
**Решение (зафиксировано):** канон — **MIT**. ISC в `package.json` — опечатка, поправить на `"license": "MIT"`.
**Релиз:** 0.9.8.

### B3. Graph не входит в 1.0
**Где:** `RareCharts.Graph` экспортируется (`src/index.js:35`), есть в таблице типов README; публичный бэклог признаёт класс сырым — «fix layout and rendering issues, improve animation» (`backlog/index.md:118`).
**Решение (зафиксировано):** Graph **выкидываем из текущих планов** — его доработка сильно после 1.0. Замораживать заведомо сырой класс в 1.0 нельзя, поэтому к 1.0 он **исключается из semver-гарантии**: помечается experimental (в README/доках), на него не распространяется обещание стабильности. Никакого rework сейчас не делаем.
**Действие к 1.0:** пометить Graph как experimental в README и таблице типов; убедиться, что он не фигурирует как «стабильный» в публичных материалах.
**Релиз:** пометка — в 0.9.9 (вместе с финализацией публичной поверхности).

### B4. Доступность
**Где:** в `rare-charts.css` нет `prefers-reduced-motion`; в SVG нет `role="img"` / `aria-label` / `<title>`/`<desc>`.
**Проблема:** позиционирование «decision-grade для финансовой отчётности» подразумевает скринридеры и корпоративный compliance.
**Минимум к 1.0:**
- уважать `prefers-reduced-motion` (гасить entry-анимации);  **[parallel]**
- давать каждому графику доступное имя (брать из `title`/`subtitle`).
**Релиз:** 0.9.9 (reduced-motion можно раньше, параллельно).

## Важно, но не строгий блокер

### N1. Версия проставляется руками в 3+ местах  **[parallel]**
**Где:** `src/index.js` (`VERSION`), баннер в трёх esbuild-скриптах `package.json`, `_data/versions.json`. В `CHANGELOG.md` уже заведён чеклист, чтобы не рассинхронить — сигнал хрупкости процесса.
**Риск:** на CDN уедет бандл с несовпадающим баннером.
**Действие:** единый источник правды + release-скрипт, стампящий версию во все места.
**Релиз:** 0.9.9.

### N2. Аудит публичного API перед заморозкой
**Где:** все 11 классов.
**Проблема:** консистентность заявлена как ценность (`charts/index.md:52`), но не проверена системно.
**Действие:** пройтись по именам опций и форме данных на единообразие; то, что меняет существующие сигнатуры (bands на TimeSeries/DualAxes, `barLabels` и т.п. из публичного бэклога), либо завести **до** заморозки, либо спроектировать строго аддитивно.
**Релиз:** 0.9.9 → финализация в 1.0.

## Что осознанно НЕ в 1.0
Новые типы и крупные фичи из публичного бэклога (`charts/backlog/index.md`): candlestick, hierarchies, diverging bar, map overlays/clickable regions, branding API, legend isolation, callouts, полноценный Graph rework. Всё аддитивно → релизы 1+.
